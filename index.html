01 · PROBLEM DEFINITION

논리식과 물리적 Layout 사이의 단절

디지털논리회로에서는 진리표와 게이트 회로를, 반도체 과목에서는 MOSFET·CMOS·Layout을 각각 학습하지만 두 영역이 실제로 어떻게 이어지는지 직접 조작하며 확인할 기회는 제한적입니다.

이 프로젝트는 논리식이 CMOS 네트워크와 물리적 레이어로 변환되는 과정을 한 화면 안의 단계형 학습 콘텐츠로 연결하는 데서 시작했습니다.

02 · LEARNING FLOW

Truth Table에서 Layout까지 이어지는 단계형 학습

  1. Truth Table
    입력 조합별 출력을 직접 설정하고 논리 동작을 확인합니다.
  2. Gate Circuit
    게이트를 배치하고 입력·출력을 연결해 논리식을 회로로 구현합니다.
  3. CMOS Schematic
    Pull-up/Pull-down Network와 pMOS/nMOS ON/OFF 동작을 시각적으로 확인합니다.
  4. Layout Lab
    N-Well, Active, Poly, Contact, Metal1을 직접 배치하고 전기적 연결을 검사합니다.
  5. AI Tutor & Report
    현재 단계에 맞는 힌트와 오답 분석, 학습 리포트 및 추천 문제를 제공합니다.

04 · LAYOUT LAB

물리적 레이어와 전기적 연결을 함께 확인

Layout Lab에서는 N-Well, P-Active, N-Active, Poly, Metal1, Contact, 출력 Y를 격자 위에 직접 배치합니다. 레이어별 지우개와 예시 답안, 정답 가이드뿐 아니라 Contact 또는 Metal 연결을 삭제했을 때 전기적 오류를 확인하는 추출 검사도 구현했습니다.

05 · AI TUTOR & LEARNING REPORT

현재 문제와 단계에 맞춘 학습 지원

AI Tutor는 현재 문제, 학습 단계, 사용자의 최근 결과를 바탕으로 힌트·오답 분석·다음 행동을 제안합니다. OpenAI API가 연결되지 않은 환경에서도 규칙 기반 fallback 답변을 제공하도록 구성했습니다.

Supabase에 저장된 점수와 이벤트를 바탕으로 강한 단계와 보완할 단계를 분석하고, 다음 문제로 바로 이동할 수 있는 학습 리포트를 제공합니다.

07 · VERIFICATION SCOPE

현재 구현 범위와 확장 방향

현재 버전은 교육 목적의 논리·CMOS·Layout 연결 검사와 단계별 학습 피드백을 구현한 웹 서비스입니다. Layout 검사는 교육용 연결 검증에 초점을 두며, 실제 EDA 수준의 DRC/LVS 전체 기능을 대체하지 않습니다.

향후에는 더 다양한 논리식과 문제 유형, 좌표 수준 Layout 진단, 관리자 분석 기능 및 모바일 사용성을 확장할 수 있습니다.
